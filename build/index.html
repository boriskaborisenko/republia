Next Generation Blockchain

Master node
Master node is a server in the decentralized network, which stores all blocks. If nodes receive a transaction, the master node gets a data transfer request.
Side chains
Sidechains is a technology that allows tokens and other digital assets from one blockchain to be securely used in a separate blockchain and then be moved back to the original blockchain if needed.
Node
Node is any computer connected to the blockchain network. Nodes interact via P2P protocols to exchange information about blocks and transactions. Nodes check if a block of transactions is valid and accept or reject it. Depending on its type, a node saves and stores part or the whole amount of blockchain data.

Safest smart-contracts

Republia Virtual Machine is implemented using functional programming language Michelson, which supports formal code verification that allows verifying smart contract before it is launched on the network. Thus, due to formal code verification, Republia VM does not execute a smart contract in case an error occurs. Moreover, smart contracts created on the Republia Smart Contract Platform are Turing complete, which ensures their correctness, because Turing completeness avoids frauds and delays in smart contracts. The smart contract, created on Republia platform, will never get caught in an endless loop because it operates under the principle of a timestamp, which ensures execution of a digital contract.

Why we are using OCaml

01
Credibility
Republia Blockchain is written in OCaml programming language. Using mathematical compiler this language rules out many errors. OCaml is applied in aircraft control software for the family of Airbus A340 and the areas where one mistake can lead to irreversible consequences. Among companies using OCaml are Microsoft, IBM, Facebook, Bloomberg, etc. Corporations choose OCaml to provide the best security of their services.
02
Safety
OCaml is a safe language. Using mathematical compiler this language rules out many programming errors. This formal verification eliminates mistakes before running the code. The compiler verifies programs before they are executed, preventing those programs from breaking down. More precisely, this protects the integrity of the data manipulated by an OCaml program.
03
Functionality
OCaml is a functional programming language: there is no restriction on the definition and use of functions. OCaml allows formal verification of individual parts of the protocol. This feature makes OCaml the perfect language to be used in open source software because in such systems there are often vulnerabilities, which participants add accidentally or intentionally.
04
Error recovery
OCaml features an exception handling mechanism to signal and handle errors and exceptional situations. Republia aims to provide zero error rate of smart contracts, because, according to the research, 34 200 out of 1 000 000 Ethereum smart contracts are potentially vulnerable.
